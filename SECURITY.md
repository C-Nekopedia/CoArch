# 安全政策

## 支持的版本

我们积极维护以下版本的安全更新：

| 版本 | 支持状态          |
|------|-------------------|
| 最新主版本 | ✅ 完全支持       |
| 上一个主版本 | ⚠️ 仅安全修复    |
| 更早版本 | ❌ 不再支持       |

## 报告漏洞

如果您发现了安全漏洞，请**不要公开报告**。请通过以下方式私下报告：

1. **发送邮件到**：[security@coarch.dev]
2. **使用 GitHub 安全报告**（如果已启用）
3. **通过维护者直接联系**（如果已知）

### 报告内容
请提供以下信息：
- 漏洞的详细描述
- 受影响的组件（前端、后端、数据库等）
- 复现步骤
- 可能的影响评估
- 建议的修复方案（可选）

## 处理流程

1. **确认收到**：我们将在 48 小时内确认收到您的报告
2. **调查评估**：我们会在 7 天内评估漏洞的严重程度
3. **修复开发**：根据漏洞严重程度，修复时间会有所不同：
   - 严重漏洞：24-72 小时内发布补丁
   - 高危漏洞：7 天内发布补丁
   - 中危漏洞：14 天内发布补丁
   - 低危漏洞：下一个常规版本中修复
4. **发布通知**：修复后，我们会发布安全公告并致谢报告者

## 安全实践

### 开发环境
- 使用最新版本的依赖包
- 定期运行 `npm audit` 检查依赖漏洞
- 启用代码安全扫描工具
- 敏感信息（密钥、密码）必须通过环境变量配置

### 部署环境
- 使用最新安全补丁的操作系统
- 配置防火墙和访问控制
- 启用 HTTPS 和安全的 HTTP 头
- 定期备份数据并测试恢复流程

### 代码安全
- 所有用户输入必须验证和清理
- 使用参数化查询防止 SQL 注入
- 实施适当的身份验证和授权检查
- 避免在客户端存储敏感信息

当前无已知安全漏洞。所有已知漏洞都会在修复后在此列出。
